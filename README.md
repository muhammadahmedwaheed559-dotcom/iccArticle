# FedHealth-CN

**A Scalable Cloud-Native Federated Learning Framework for Privacy-Preserving Clinical Mortality Prediction**

> IEEE ICC 2026 | M. Ahmed Waheed · Mehvish Shahid · Fahad Amir
> Department of Computer Science, University of Central Punjab (UCP), Lahore, Pakistan

---

## 📋 Overview

FedHealth-CN is a cloud-native federated learning system that predicts 48-hour ICU mortality
from MIMIC-III clinical data across ten simulated hospital nodes — without any raw patient data
leaving its origin container.

It addresses two core bottlenecks of standard Federated Averaging (FedAvg):

| Problem | FedHealth-CN Solution |
|---|---|
| **Straggler Problem** | Asynchronous aggregation — server does not wait for slow nodes |
| **Non-IID Data** | Quality-weighted aggregation — `weight = n_samples / (1 + local_loss)` |

## 🚀 Quick Start (2 commands)

```bash
# Step 1 — Preprocess raw MIMIC-III CSVs → masterdataset.csv + hospital shards
python preprocess.py

# Step 2 — Train all models + generate all 6 IEEE figures + print Table III values
python evaluate.py
```

That is all. `evaluate.py` is fully self-contained:
- If no saved models exist → it trains Centralized, FedAvg, and FedHealth-CN from scratch
- If saved models exist → it loads them and skips training
- Either way → all 6 figures are generated and Table III values are printed to console

## 📂 Script Details

### `preprocess.py`

| Task | Detail |
|---|---|
| Input | `data/mimic/ADMISSIONS.csv`, `PATIENTS.csv`, `LABEVENTS.csv`, `CPTEVENTS.csv` |
| Output | `data/mimic/masterdataset.csv` + `data/shards/hospital_N.csv` (N = 1–10) |
| Target label | `hospital_expire_flag` (1 = died within 48 h, 0 = survived) |
| Features | 22 features — age, gender, LOS, lab values, vitals, admission type |
| Sharding | Non-IID split: heterogeneous by shard size and mortality rate |

### `evaluate.py`

| Step | What happens |
|---|---|
| 1 | Loads `masterdataset.csv` + all hospital shards |
| 2 | Trains **Centralized MLP** (80 epochs, Adam, BCE loss) — or loads saved |
| 3 | Runs **FedAvg** simulation (50 rounds, 5 local epochs/round) — or loads saved |
| 4 | Runs **FedHealth-CN** (50 rounds, quality-weighted async aggregation) — or loads saved |
| 5 | Computes probabilities from all 3 models on the global test set |
| 6 | Generates **6 IEEE-standard figures** as PDF + PNG |
| 7 | Prints **Table III** values (Accuracy, Precision, Recall, F1) ready to paste into LaTeX |

## 📐 Model Architecture

All three models use the same MLP:

```
Input (22 features)
  → Linear(22 → 32) + BatchNorm1d + ReLU + Dropout(0.3)
  → Linear(32 → 16) + ReLU
  → Linear(16 → 1)  + Sigmoid
  → Output: mortality probability ∈ [0, 1]
```

### Aggregation Algorithms

**FedAvg (baseline)**
```
w_global = Σ (n_k / N) × w_k
```
Each client contributes proportional to its sample count. All clients must finish
before the server aggregates (synchronous barrier).

**FedHealth-CN (proposed)**
```
quality_k  = n_k / (1 + local_loss_k)
w_global   = (1 - η) × w_global + η × Σ (quality_k / Σ quality) × w_k
```
Clients with lower local loss receive higher weight. Server aggregates
asynchronously without waiting for stragglers. η = 0.1 (server learning rate).

## 🔬 Dataset

**MIMIC-III** (Medical Information Mart for Intensive Care, v1.4)
- Source: Beth Israel Deaconess Medical Center, Boston, MA
- Access: [https://physionet.org/content/mimiciii/](https://physionet.org/content/mimiciii/) (requires PhysioNet credentialing)
- Tables used: `ADMISSIONS`, `PATIENTS`, `LABEVENTS`, `CPTEVENTS`
- Target: 48-hour in-hospital mortality (`hospital_expire_flag`)

> ⚠️ MIMIC-III data is de-identified but access-controlled.
> Never commit raw CSV files to a public repository.

## 🔒 Privacy Design

- Raw EHR data **never leaves** its local shard/container
- Only serialized model weight tensors are transmitted
- Kubernetes network policy blocks inter-node direct communication
- Compliant with **HIPAA** (minimum necessary standard) and **GDPR** (Article 5(1)(c) data minimization)
